## Role Taxonomy: What Each Title Means

### Understanding Role Titles

| Role | Focus | Typical Skills | Who Hires |
|------|-------|---------------|-----------|
| **Research Scientist** | Publish papers, advance SOTA, design new architectures | PhD preferred, strong math, publications | Google, Meta, MSR, DeepMind, OpenAI |
| **Applied Scientist** | Solve business problems with ML research | MS/PhD, ML theory + engineering | Amazon (primary title), Microsoft, LinkedIn |
| **AI/ML Scientist** | Bridge research and product — design novel ML solutions | MS/PhD, deep ML knowledge | Startups, mid-size companies |
| **ML Engineer** | Build, deploy, optimize ML systems in production | Strong coding, MLOps, system design | All companies |
| **Data Scientist** | Analytics + modeling, business insights, light ML | Statistics, SQL, Python, business acumen | All companies (broadest category) |
| **AI Engineer** | Build AI-powered products using existing models (APIs, RAG, agents) | Software engineering, LLM APIs, prompt engineering | Startups, product companies |
| **NLP Engineer / Scientist** | Specialize in language tasks | NLP-specific knowledge, transformers | Companies with text-heavy products |
| **Computer Vision Engineer** | Specialize in image/video tasks | CNN, detection, segmentation, diffusion | Manufacturing, autonomous driving, retail |
| **MLOps Engineer** | ML infrastructure, deployment, monitoring | Kubernetes, CI/CD, model serving | Large companies with ML teams |
| **Quantitative Researcher** | ML for financial markets | Strong math, statistics, ML, finance | Banks, hedge funds, trading firms |

## Application Strategy & Tips

### The Application Funnel

```
100 Applications
     │
     ▼
 20 Recruiter Screens (20% response rate with referrals)
     │
     ▼
 10 Technical Phone Screens
     │
     ▼
  4 Onsite / Virtual Loops
     │
     ▼
  1-2 Offers

Timeline: 2-4 months from start to offer
```

### Strategy by Role Type

**For Research Scientist roles**:
- Publications matter (even 1-2 workshop papers help)
- Strong GitHub with research implementations
- Target: Google Research, MSR India, Meta FAIR, Amazon Science
- Network at conferences: NeurIPS, ICML, ACL, EMNLP

**For Applied Scientist / ML Engineer roles**:
- Strong DSA + ML coding skills
- System design for ML (serving, scale, data pipelines)
- Target: Amazon (biggest hirer of Applied Scientists), Microsoft, Flipkart
- LeetCode (Medium/Hard) + ML system design prep

**For Data Scientist roles**:
- Statistics + SQL + Python + business sense
- Case studies and A/B testing knowledge
- Target: Widest range of companies
- Practice product sense questions + ML fundamentals

**For AI Engineer roles (emerging)**:
- LLM API expertise, RAG, agents, prompt engineering
- Full-stack development + AI integration
- Target: Startups, product companies building AI features
- Build visible projects (open-source, demos, blog posts)

### Networking That Works in India

1. **LinkedIn**: Follow and engage with AI leaders at target companies. Indian AI community is very active on LinkedIn
2. **Kaggle**: Competitions + notebooks build visible credibility
3. **Twitter/X**: Follow AI researchers, engage in technical discussions
4. **Conferences**: NeurIPS, ICML, local meetups (Bangalore ML, AI Saturday)
5. **College alumni networks**: IIT/IISc/IIIT/NIT alumni networks are powerful referral channels
6. **Company referrals**: The #1 way to get interviews at FAANG in India — ask connections directly

### Resume Tips for India AI Market

```
DO:
✓ Quantify impact ("Improved model accuracy by 15%, saving $2M/year")
✓ List specific technologies (PyTorch, HuggingFace, LangChain, vLLM)
✓ Mention scale ("Served 10M users", "Processed 1TB/day")
✓ Include publications, patents, open-source contributions
✓ Keep to 1-2 pages

DON'T:
✗ List every technology you've ever touched
✗ Use vague descriptions ("worked on ML projects")
✗ Include photos, personal details (for global companies)
✗ Ignore ATS — use standard section headers
```
